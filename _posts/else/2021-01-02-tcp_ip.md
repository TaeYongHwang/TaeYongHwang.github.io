# 진행 순서
- 운영체제별로 다루는 방식이 다르다. (리눅스의 경우 소켓도 파일로 취급, 윈도우에서 별개로 취급 등)

## 서버 소켓
1. 소켓 생성
2. 해당 소켓에 IP, PORT 바인딩 (BIND)
3. 연결 가능한 상태로 변경 (LISTEN)
4. 연결 요청에 대한 수락 (ACCEPT)

## 클라이언트 소켓
1. 소켓 생성
2. 해당 소켓에 서버 연결 (CONNECT)
